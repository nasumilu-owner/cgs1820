Web Map Service (WMS)

Introduction

The primary responsibility of a web map server is to produce georeferenced images from geographic information. The OGC first released the WMS standard, version 1.0.0 on April 19, 2000. Version 1.3.0 is the current standard which defines three operations GetCapabilities, GetMap, and GetFeatureInfo.

Assumptions

It is assumed that you have some basic understanding of web services and the OWS paradigm. Additional information on this can be found on the OGC website.

Operations

GetCapabilities

The mandatory GetCapabilities operation provides metadata [machine readable information] describing the server's capabilities. There are two required parameters SERVICE & REQUEST. The SERVICE parameter must have a value of WMS & the REQUEST must have a value of GetCapabilities.

Parameter Description Required Value
VERSION The requested Version —
SERVICE The request service type WMS
REQUEST The requested service operation. GetCapabilities
FORMAT Output format of the service metadata response —
UPDATESEQUENCE Sequence number or string for cached control —
= Yes = No

GetMap

The mandatory GetMap operation is used to retrieved the georeferenced image [map] or a service exception when an error occured. Similar to the GetCapabilities operation the GetMap has the following parameters:

Parameter Description Required Value
VERSION The requested service version; A value of "1.3.0" must be used for GetMap requests to comply with ISO 19128. 1.0.0 | 1.3.0
REQUEST The requested service operation. GetMap
LAYERS A comma-separated list of requested map layers —
STYLES A comma-separated list of one style per requested layer —
CRS The coordinate reference system for which the BBOX parameter utilizes —
BBOX The requested min. & max. x & y-coordinate value —
WIDTH The georeference map image's width in pixels —
HEIGHT The georeference map image's height in pixels —
FORMAT Indicates the desired image output format —
TRANSPARENT Indicate whether the map image should have a transparent background. true | false
BGCOLOR Hexadecimal color value fot he background color #0xFFFFFF
EXCEPTION Indicates the format in which exceptions are to be reported by the service —
TIME Time value of layer desired —
ELEVATION Elevation of the layer desired —
= Yes = No

Example of the GetMap operation using HTTP GET method:

https://map.nasumilu.com/bof/wms?service=WMS&version=1.1.0&request=GetMap&layers=bof%3Abellamyoaksfarm&bbox=-9192698.0%2C3486522.500000001%2C-9187783.0%2C3494379.249999998&width=480&height=768&srs=EPSG%3A3857&format=image/png   Back to Top

GetFeatureInfo

This optional operation is design to provide clients with a capabiltiy of obtaining information about specific features in the map's picture. When the user clicks onto the georeferenced image the coordinate of the click in reference to the image (top, left) is sent to the WMS to be translated and query the spatial data for additional information.

Parameter Description Required Value
VERSION The requested service version 1.3.0
REQUEST The service requsted GetFeatureInfo/td>
QUERY_LAYER Comma separated list of one or more layers to query. —
INFO_FORMAT The return format of the feature's info —
FEATURE_COUNT Indicates the total number of feature's info to return 1
I The i-coordinate i pixels of the feature defined by the map's coordinate system. Integer value between 0 & the maximum value of the i axis
J The j-coordinate in pixels of the feature defined by the map's coordinate system. Integer value between 0 and the maximum value of the j axis
EXCEPTION Indicates the format in which exceptions are to be reported by the service —
= Yes = No

Example of the GetFeatureInfo operation using HTTP GET method:

https://map.nasumilu.com/bof/wms?SERVICE=WMS&VERSION=1.3.0&REQUEST=GetFeatureInfo&FORMAT=image%2Fpng&TRANSPARENT=true&QUERY_LAYERS=bof%3Abellamyoaksfarm&LAYERS=bof%3Abellamyoaksfarm&INFO_FORMAT=application/vnd.ogc.gml&I=50&J=50&CRS=EPSG%3A3857&STYLES=&WIDTH=101&HEIGHT=101&BBOX=-9190117.55903557%2C3491254.059947751%2C-9189635.590438515%2C3491736.0285448045   Back to Top
