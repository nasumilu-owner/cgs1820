Web Feature Service (WFS)

Introduction

The web feature service (WFS) is an International Standard which defines eleven operations. This standard allows clients the ability to create, modify, and exchange geographical information via the Internet. The feature data is exchanged utilizing Geography Markup Language (GML).

To understand a WFS it is important to understand what constitutes a feature. A feature is the spatial representation of some thing in coordinate space. Features can be either a point, line, polygon, or a collection. Homogeneous features that share a common set of attributes are commonly called a feature class.

Assumptions

It is assumed that you have some basic understanding of web services and the OWS paradigm. Additional information on this can be found on the OGC website.

Operations

GetCapabilities

The mandatory GetCapabilities operation provide metadata [machine readable information] describing the server's capabilities. There are two required parameters SERVICE & REQUEST. The SERVICE parameter must have a value of WFS & the REQUEST must have a value of GetCapabilities.

Parameter Description Required Value
SERVICE The request service type Yes WFS
REQUEST The requested service operation. Yes GetCapabilities
VERSION The requested Version No 1.0.0 | 1.1.0 | 2.0.0

Example of the GetCapabilities operation using HTTP GET method:

https://map.nasumilu.com/ows?service=wfs&version=2.0.0&request=GetCapabilities   Back to Top

DescribeFeatureType

The mandatory DescribeFeatureType operation is used to describe the structure of any feature type available to the server. In other words, it provides the client with information about the features spatial reference system, geometry type, and properties as an XML schema definition (XSD).

Parameter Description Required Value
SERVICE The request service type Yes WFS
REQUEST The requested service operation. Yes DescribeFeatureType
VERSION The requested Version No 1.0.0 | 1.1.0 | 2.0.0
TYPENAMES A comma-separated list of feature type names Yes —
OUTPUTFORMAT The format for which the server should return the feature description information No —

Example of the DescribeFeatureType operation using HTTP GET method:

https://map.nasumilu.com/wfs?service=wfs&version=2.0.0&request=DescribeFeatureType&typeNames=bof:parcel   Back to Top

GetFeature

The GetFeature operation is used to get feature instances from the WFS data store, either spatially or non-spatially. The client should also be able to specify which of the feature's properties to retieve. This operation is very flexible to allow the client to retrieve features in a variety of was, as such below are a few of the more common ways to obtain feature(s) from a WFS.

Example 1 - Retrieve the maximum allowed features

This example will retrieve the maximum number of features allowed by the WFS. 

    
    https://map.nasumilu.com/wfs?service=wfs
        &version=2.0.0
        &request=GetFeature
        &typeNames=bof:parcel
Example 2 - Limit results to 5 features

This example will retrieve up to 5 features from the WFS. 

    
    https://map.nasumilu.com/wfs?service=wfs
        &version=2.0.0
        &request=GetFeature
        &typeNames=bof:parcel
        &count=5

Or for WFS version 1.0.0 or 1.1.0 use the maxFeatures parameter.

    
        https://map.nasumilu.com/wfs?service=wfs
        &version=1.1.0
        &request=GetFeature
        &typeNames=bof:parcel
        &maxFeatures=5

Example for WFS version 1.0.0

    
        https://map.nasumilu.com/wfs?service=wfs
        &version=1.0.0
        &request=GetFeature
        &typeNames=bof:parcel
        &maxFeatures=5
